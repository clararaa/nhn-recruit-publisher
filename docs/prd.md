# NHN 채용공고 퍼블리셔 — Product Requirements Document (PRD)

## Goals and Background Context

### Goals
- 채용 공고 외부 플랫폼 게재 시간을 1시간 → 10분 이내로 단축
- 10개 플랫폼에서 서식 깨짐 없는 동일한 품질의 공고문 보장
- 채용팀 7명 전원이 별도 교육 없이 즉시 사용 가능

### Background Context
NHN 채용팀은 자사 채용사이트(careers.nhn.com) 공고를 사람인, 잡코리아, 링크드인 등 최대 10개 외부 채용 플랫폼에 게재하고 있다. 플랫폼마다 에디터가 달라 복사/붙여넣기 시 서식이 깨지며, 월 20~30건 × 최대 1시간의 반복 작업이 발생한다. NHN은 다수의 관리법인(NHN, NHN Cloud, NHN Dooray 등)을 운영하므로, 법인별 상단 배너 이미지는 제외하고 텍스트 콘텐츠만 변환하는 도구가 필요하다.

## Requirements

### Functional

- **FR1**: 사용자가 텍스트 영역에 채용 공고 내용을 붙여넣을 수 있어야 한다
- **FR2**: 붙여넣은 텍스트에서 표준 공고 섹션(회사/팀 소개, 주요업무, 자격요건, 우대사항, 지원방법, 참고사항, 합류여정 등)을 자동 인식하여 구조화해야 한다
- **FR9**: 원본 공고의 볼드 처리된 텍스트(섹션 제목, 강조 키워드 등)를 감지하여 생성된 HTML에 `<strong>` 태그로 반영해야 한다
- **FR10**: 공고문 상단에 배너 이미지를 선택적으로 추가할 수 있어야 한다. 기본 이미지가 설정되어 있되, 공고별로 다른 이미지를 업로드하여 교체할 수 있어야 한다
- **FR3**: 구조화된 내용을 인라인 스타일이 적용된 HTML로 변환하고, 원클릭으로 클립보드에 복사할 수 있어야 한다
- **FR4**: 동일한 내용을 이미지(PNG)로 변환하여 다운로드할 수 있어야 한다
- **FR5**: 생성된 공고문의 실시간 미리보기를 제공해야 한다
- **FR6**: 공고문 너비는 ~700px 기준의 세로형 레이아웃이어야 한다
- **FR7**: 사용자가 섹션 인식 결과를 수동으로 수정(섹션 제목 변경, 순서 변경, 추가/삭제)할 수 있어야 한다
- **FR8**: NHN 스타일(블랙 & 화이트, 미니멀, 볼드 타이포)의 디자인이 적용되어야 한다

### Non Functional

- **NFR1**: 공고 텍스트 붙여넣기 후 HTML/이미지 생성까지 3초 이내 완료
- **NFR2**: Vercel 무료/Pro 플랜 범위 내에서 운영 가능해야 한다
- **NFR3**: Chrome 최신 버전에서 정상 동작해야 한다
- **NFR4**: 생성된 인라인 HTML이 10개 대상 플랫폼 에디터에서 서식 깨짐 없이 렌더링되어야 한다
- **NFR5**: 클라이언트 사이드에서 모든 처리가 완료되어야 한다 (서버 의존 없음)

## User Interface Design Goals

### Overall UX Vision
채용담당자가 3단계(붙여넣기 → 확인 → 복사/다운로드)로 작업을 완료할 수 있는 심플한 단일 페이지 도구

### Core Screens and Views
- **메인 화면 (단일 페이지)**: 좌측 입력 영역 + 우측 미리보기 영역의 2컬럼 레이아웃

### Branding
- NHN Careers 디자인 톤: 블랙 & 화이트 기반, 볼드 영문 타이포, 미니멀
- 상단 법인별 배너 이미지 제외 (텍스트 콘텐츠만)

### Target Platforms
- Web (Desktop Chrome)

## Technical Assumptions

### Repository Structure: Monorepo (단일 프로젝트)

### Service Architecture
- 프론트엔드 전용 SPA (백엔드 없음)
- 모든 처리는 클라이언트 사이드

### Testing Requirements
- 핵심 기능(섹션 파싱, HTML 생성) 단위 테스트

### Additional Technical Assumptions
- Next.js + Vercel 배포
- html-to-image 또는 html2canvas 라이브러리로 이미지 생성
- 인라인 스타일 HTML 생성은 자체 변환 로직으로 구현
- Tailwind CSS로 UI 스타일링
- TypeScript 사용

## Epic 1: 프로젝트 기반 구축 및 텍스트 입력/파싱

프로젝트 초기 세팅과 핵심 입력/파싱 기능을 완성하여, 사용자가 공고 텍스트를 붙여넣으면 구조화된 섹션으로 분리되는 것을 확인할 수 있다.

### Story 1.1: 프로젝트 초기 세팅

As a 개발자,
I want Next.js 프로젝트가 Vercel 배포 가능한 상태로 세팅되기를,
so that 이후 기능 개발의 기반이 마련된다.

**Acceptance Criteria:**
1. Next.js + TypeScript + Tailwind CSS 프로젝트 생성
2. Vercel 배포 설정 완료 (빌드 성공)
3. 기본 레이아웃(헤더 + 2컬럼 구조) 구현
4. NHN 스타일 기본 테마(블랙 & 화이트, 볼드 타이포) 적용

### Story 1.2: 텍스트 입력 영역 구현

As a 채용담당자,
I want 공고 내용을 텍스트 영역에 붙여넣을 수 있기를,
so that 공고문 변환 작업을 시작할 수 있다.

**Acceptance Criteria:**
1. 좌측 패널에 텍스트 입력 영역(textarea) 표시
2. 붙여넣기(Ctrl+V) 동작 정상 지원
3. 입력된 텍스트 실시간 상태 관리
4. 입력 영역 비우기(초기화) 버튼 제공

### Story 1.3: 공고 섹션 자동 파싱

As a 채용담당자,
I want 붙여넣은 텍스트가 자동으로 섹션별로 분리되기를,
so that 구조화된 공고문을 바로 확인할 수 있다.

**Acceptance Criteria:**
1. 표준 섹션 키워드(회사소개/팀소개, 주요업무, 자격요건, 우대사항, 지원방법, 참고사항, 합류여정 등) 기반 자동 파싱
2. 파싱 결과를 섹션 제목 + 본문 리스트로 구조화
3. 불릿 포인트(•, -, · 등)를 HTML 리스트로 변환
6. 볼드 처리된 텍스트(섹션 제목, 괄호 안 부제목 등)를 감지하여 `<strong>` 태그로 보존
4. 파싱 결과를 우측 미리보기 영역에 실시간 표시
5. 인식되지 않는 텍스트는 "기타" 섹션으로 분류

## Epic 2: HTML 생성 및 미리보기

파싱된 공고 내용을 NHN 스타일의 인라인 HTML로 변환하고, 미리보기와 원클릭 복사 기능을 제공한다.

### Story 2.1: 인라인 스타일 HTML 생성

As a 채용담당자,
I want 파싱된 공고가 인라인 스타일 HTML로 변환되기를,
so that 외부 플랫폼 에디터에 붙여넣어도 서식이 깨지지 않는다.

**Acceptance Criteria:**
1. 모든 스타일이 인라인(style 속성)으로 적용된 HTML 생성
2. 외부 CSS/클래스 의존 없는 자체 완결형 HTML
3. 700px 너비 기준 레이아웃
4. NHN 스타일 적용: 블랙 & 화이트 톤, 볼드 섹션 제목, 깔끔한 불릿 리스트
5. 섹션 간 적절한 여백과 구분선

### Story 2.2: 미리보기 및 클립보드 복사

As a 채용담당자,
I want 생성된 HTML을 미리보기로 확인하고 원클릭으로 복사하기를,
so that 바로 외부 플랫폼에 붙여넣을 수 있다.

**Acceptance Criteria:**
1. 우측 패널에 생성된 HTML의 렌더링된 미리보기 표시
2. "HTML 복사" 버튼 클릭 시 인라인 HTML이 클립보드에 복사
3. 복사 성공 시 피드백(토스트 메시지 등) 표시
4. 미리보기가 실제 외부 플랫폼에서 보이는 것과 동일하게 렌더링

## Epic 3: 이미지 생성 및 마무리

이미지 다운로드 기능과 섹션 수동 편집 기능을 추가하여 MVP를 완성한다.

### Story 3.1: 이미지(PNG) 생성 및 다운로드

As a 채용담당자,
I want 공고문을 이미지(PNG)로 다운로드하기를,
so that 이미지만 지원하는 플랫폼에도 게재할 수 있다.

**Acceptance Criteria:**
1. "이미지 다운로드" 버튼 클릭 시 미리보기와 동일한 디자인의 PNG 파일 생성
2. 파일명에 공고 제목 또는 날짜 포함
3. 이미지 품질: 웹 게재에 적합한 해상도 (2x 레티나 대응)
4. 생성 시간 3초 이내

### Story 3.2: 섹션 수동 편집

As a 채용담당자,
I want 자동 파싱된 섹션을 수동으로 수정하기를,
so that 파싱이 완벽하지 않을 때 직접 조정할 수 있다.

**Acceptance Criteria:**
1. 섹션 제목 인라인 편집 가능
2. 섹션 순서 드래그앤드롭 변경 가능
3. 섹션 추가/삭제 가능
4. 수정 시 미리보기 실시간 반영
